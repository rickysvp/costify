# Costify 消息通知类型规范

## 一、预算与费用类 (Budget & Billing)

| 类型 | 触发条件 | 严重程度 | 通知方式 |
|------|----------|----------|----------|
| 预算使用达 80% | 项目/API Key 预算使用达到 80% | 警告 | 应用内+邮件 |

## 通知方式说明

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **应用内** | 消息中心显示，有未读红点 | 所有通知 |
| **邮件** | 发送到注册邮箱 | 重要通知、日报周报 |
| **短信** | 发送到绑定手机 | 严重告警、安全事件 |
| **Webhook** | 推送到用户配置的 URL | 实时事件、集成场景 |
| **Slack** | 推送到 Slack 频道 | 团队协作场景 |

---

## 严重程度定义

| 级别 | 说明 | 响应要求 |
|------|------|----------|
| **信息** | 正常业务流程通知 | 无需立即处理 |
| **警告** | 需要注意，可能有问题 | 建议 24 小时内查看 |
| **严重** | 需要立即处理的问题 | 需要立即查看处理 |

## 消息格式规范

### 标题格式
```
[项目名] 事件描述
```

### 内容格式
```
具体描述 + 数据详情 + 操作建议
```

### 示例
```
标题: [ChatBot Pro] 项目已暂停 - 预算耗尽
内容: 项目 "ChatBot Pro" 已达到月度预算上限 $500.00，API 调用已暂停。
      如需恢复服务，请调整预算或等待下月重置。
操作: [调整预算] [查看详情]
```
