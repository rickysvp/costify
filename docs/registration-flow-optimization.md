# 注册流程优化 - 邀请成员业务链条

> 版本: v1.0.1
> 需求编号: REQ-003
> 优先级: P0
> 状态: 计划中
> 更新日期: 2026-04-28

---

## 一、需求概述

优化 AnyTokn 的注册与邀请流程，实现完整的成员邀请业务链条。支持组织管理员邀请新成员，被邀请人通过邮件链接完成注册或登录后自动加入组织。

---

## 二、业务目标

1. **降低成员加入门槛** - 被邀请人只需点击邮件链接即可完成加入组织
2. **支持两种用户状态** - 已注册用户直接登录加入，未注册用户引导注册后自动加入
3. **提升邀请转化率** - 简化操作流程，减少用户流失
4. **确保安全性** - 邀请链接带有时效性 token，防止恶意使用

---

## 三、用户故事

### 故事 1: 管理员发起邀请
```
作为 组织管理员
我希望 输入成员邮箱并发送邀请邮件
以便 新成员能够收到邀请并加入组织
```

### 故事 2: 已注册用户接受邀请
```
作为 已注册的 AnyTokn 用户
我希望 点击邀请链接后直接登录并自动加入新组织
以便 无需手动申请加入
```

### 故事 3: 未注册用户接受邀请
```
作为 尚未注册 AnyTokn 的用户
我希望 点击邀请链接后填写密码完成注册，然后自动加入组织
以便 一站式完成注册和加入
```

## 六、功能详细设计

### 6.1 发起邀请流程

**入口**: 成员管理页面 (`/members`)

**交互步骤**:
1. 管理员点击"邀请成员"按钮
2. 弹出邀请弹窗
3. 输入被邀请人邮箱
4. 输入被邀请人姓名
5. 选择角色（管理员 / 成员）
6. 选择要分配的项目（可多选）
7. 点击"发送邀请"
8. 显示发送成功提示（含临时密码）

### 6.3 邀请处理页 (`/invite`)

**功能**: 中间处理页，验证邀请token并判断用户状态

**流程**:
1. 页面加载时解析 URL 中的 `token` 参数
2. 调用 API 验证 token 有效性
3. 根据返回结果处理：
   - Token 无效/过期 → 显示错误信息
   - Token 有效 + 用户已登录 → 自动接受邀请 → 跳转 Dashboard
   - Token 有效 + 用户未登录 → 判断邮箱是否已注册 → 跳转登录或注册页

**页面状态**:
```
┌─────────────────────────────────────┐
│                                     │
│      [Loading Spinner]              │
│      正在处理邀请...                │
│                                     │
└─────────────────────────────────────┘
```

## 七、API 接口设计

### 7.1 创建邀请
```
POST /api/invitations

Request:
{
  "emails": ["user1@example.com", "user2@example.com"],
  "role": "member",
  "message": "欢迎加入我们的团队",
  "organization_id": 1
}

Response:
{
  "success": true,
  "data": {
    "sent": 2,
    "failed": 0,
    "invitations": [
      {
        "id": 1,
        "email": "user1@example.com",
        "token": "abc123",
        "expires_at": "2026-05-05T12:00:00Z"
      }
    ]
  }
}
```

### 7.2 验证邀请 Token
```
GET /api/invitations/:token/validate

Response:
{
  "success": true,
  "data": {
    "valid": true,
    "email": "user@example.com",
    "organization": {
      "id": 1,
      "name": "技术研发部"
    },
    "inviter": {
      "name": "张三",
      "email": "zhangsan@example.com"
    },
    "role": "member",
    "expires_at": "2026-05-05T12:00:00Z"
  }
}
```

### 7.3 接受邀请（已注册用户）
```
POST /api/invitations/:token/accept

Headers:
Authorization: Bearer <token>

Response:
{
  "success": true,
  "data": {
    "message": "已成功加入组织",
    "organization": {
      "id": 1,
      "name": "技术研发部"
    }
  }
}
```

### 7.4 注册并加入（新用户）
```
POST /api/invitations/:token/register

Request:
{
  "username": "新用户",
  "password": "password123"
}

Response:
{
  "success": true,
  "data": {
    "token": "jwt_token_here",
    "user": {
      "id": 1,
      "email": "user@example.com",
      "username": "新用户"
    },
    "organization": {
      "id": 1,
      "name": "技术研发部"
    }
  }
}
```

## 九、技术实现要点

### 9.1 邀请 Token 生成
- 使用加密安全的随机字符串（如 crypto.randomBytes）
- 长度建议 32-64 字符
- 设置过期时间（默认 7 天）

### 9.2 邮件服务集成
- 推荐方案：Resend / SendGrid / AWS SES
- 需要配置邮件模板
- 支持 HTML 格式邮件

### 9.3 安全考虑
- 邀请链接只能使用一次
- Token 过期后自动失效
- 防止暴力破解（限制验证频率）
- 邀请邮件中的邮箱需脱敏显示

### 9.4 错误处理
| 错误场景 | 处理方式 |
|----------|----------|
| Token 无效 | 显示"邀请链接无效或已过期" |
| Token 已使用 | 显示"该邀请已被使用" |
| 用户已加入 | 显示"你已经是该组织成员" |
| 邮箱不匹配 | 显示"请使用被邀请的邮箱登录" |

## 十、验收标准

### 功能验收
- [ ] 管理员可以成功发起邀请
- [ ] 被邀请人可以收到邀请邮件
- [ ] 已注册用户点击链接可以登录并自动加入
- [ ] 未注册用户点击链接可以注册并自动加入
- [ ] 过期/无效链接显示友好错误提示
- [ ] 已加入组织的用户再次点击显示已加入提示

### 性能验收
- [ ] 邀请邮件发送延迟 < 5 秒
- [ ] 邀请验证 API 响应 < 500ms
- [ ] 页面加载时间 < 2 秒

### 安全验收
- [ ] Token 不可预测
- [ ] Token 过期后无法使用
- [ ] 邀请只能被接受一次
- [ ] 邮箱不匹配时无法加入
